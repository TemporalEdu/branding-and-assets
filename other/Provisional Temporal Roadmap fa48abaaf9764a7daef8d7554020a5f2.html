Provisional Temporal Roadmap

Phase 1 (Complete in Q1/Q2 2023 Ideally)

  • Build a Minimum viable product (Engineering / Product)
    • A backend that can handle a decent amount of people
    • Assets Branding
    • Web Frontend v1
    • Continue the mission of this project by keeping Temporal effective, rewarding, accessible
    • Build a cloud for the grade book where the information will be handled securely
  • Finance
    • Software as a Service-based model
    • Software audited by professional cybersecurity and accounting firms
    • Evaluate our Technology and Idea with Students, Teachers, and School Districts
    • Explore initial funding for the project from investors
  • Marketing
    • Launch our website with testers
    • Social Media Presence on [Github, Twitter, Discord, Reddit, and Medium]
    • Create tutorials for our software
  • Operations
    • Begin building a target user group for Temporal (Schools, Teachers, Students, and Families) using digital learning software while being accessible for anyone to use, anywhere.
    • Create a committee using candidates from the targeted user groups who will provide input.
    • Begin building contracts with School Districts and partnerships with Teachers

Phase 2 (Complete in Q4/2023 Q1/2024 Ideally)

  • Continue building our product (Web Frontend/Backend V2)
    • Incorporate web3-based technologies into Temporal’s software
      1. Use artificial intelligence and augment human intelligence to calculate the best learning methods for a specific student.
      1. Build an advanced frontend tool for viewing big data at any time and anywhere while optimizing efficiency and speed.
      1. Allow anyone to create apps and extensions for Temporal which will allow for greater utility and control
      1. Allow schools to create smart contracts using our software which will make the software more secure and customizable and allow for more integration / cross functionality
      1. Using Edge Computing / Decentralized Hosting in order to self-host the Backend of the Website and other operations which will allow the backend to handle more data

    (Priority sorted by numbers)
